# ADISE25_2020122
ΞΕΡΗ-Εφαρμογή 2020122

   Η συγκεκριμένη web εφαρμογή αποτελεί την
υλοποίηση του Ελληνικού παιχνιδιού 'Ξερή'.Είναι ενα 2player παιχνίδι το οποίο είχε ως βασική απαίτηση την υλοποίηση του με την χρήση Web API και human vs human.

Περιγραφή Εφαρμογής
    Αρχικά ο χρήστης που επισκέφτεται την
εφαρμογή αντικρίζει ενα welcome screen το οποίο του ζητάει να εισάγει το username του,πρέπει να είναι μοναδικό,δηλαδή να μην υπάρχει στην βάση δεδομένων.Αντίστοιχα το κουμπί της σύνδεσης έχει και την ικανότητα έιτε να εισάγει νέους χρήστες στην βάση είτε να κάνει την ταυτοποίηση του username που τοποθετείται από τον χρήστη.

    Στην δεύτερη σελίδα μετά την επιτυχή σύνδεση,ο χρήστης έχει
τρεις επιλογές.Η πρώτη είναι η δημιουργία παιχνιδιού όπου του παραδίδεται ένα μοναδικό ID(αριθμός ID κλειδιών είναι από 0-10.000) το οποίο πρέπει να το μοιραστεί με τον αντίπαλο του.Η δεύτερη επιλογή είναι να εισάγει ένα αριθμό ID.Σε περίπτωση που δεν υπάρχει παιχνίδι με το συγκεκριμένο αριθμό βγαίνει το αντίστοιχο μήνυμα λάθους,αλλιώς μπαίνει στο παιχνίδι με αυτό τον μοναδικό αριθμό.Η τελευταία επιλογή είναι το κουμπί logout όπου ο χρήστης αποσυνδέεται απο την εφαρμογή.

    Στο τρίτο στάδιο είναι η υλοποίηση του παιχνιδιού.Το
παιχνίδι ξεκινά με την αρχικοποίηση μιας τράπουλας 52 φύλλων, η οποία ανακατεύεται και μοιράζεται δυναμικά. Η εφαρμογή ενσωματώνει προηγμένη λογική για:

Αναγνώριση Σειράς: Το API εμποδίζει κινήσεις εκτός σειράς, διατηρώντας τον συγχρονισμό μεταξύ των παικτών.

Υπολογισμό Σκορ: Καταγράφονται αυτόματα οι πόντοι από κάρτες αξίας (Άσοι, Ρήγες, κ.λπ.), οι ειδικοί πόντοι (10♦, 2♣), καθώς και οι πόντοι από «Ξερές».

Αναγνώριση Deadlock: Το σύστημα ανιχνεύει αυτόματα τον τερματισμό του παιχνιδιού όταν εξαντληθούν τα φύλλα και αποδίδει το bonus των +3 πόντων στον παίκτη με τα περισσότερα μαζεμένα φύλλα.

Live Log και Εγκατάλειψη: Παρέχεται ένα ιστορικό κινήσεων σε πραγματικό χρόνο, ενώ υποστηρίζεται η λειτουργία εγκατάλειψης (forfeit) με άμεση ενημέρωση του αντιπάλου.


    Κατά την διάρκεια του παιχνιδιού υπάρχει και ένα ιστορικό το
οποίο λέει ποιες κάρτες έχουν παιχτεί και από ποιον.Το παιχνίδι τερματίζει όταν τελειώσει η τράπουλα και ο χρήστης με το μεγαλύτερο σκορ είναι ο νικητής.Ακόμη σε περίπτωση εγκατάλειψης η εφαρμογή περιλαμβάνει μηχανισμό ασφαλούς εξόδου. Όταν ένας παίκτης επιλέξει "Εγκατάλειψη", το σύστημα ζητά επιβεβαίωση μέσω JavaScript alert. Με την επιβεβαίωση, το API ενημερώνει τη βάση δεδομένων, θέτει το status σε "ended" και ενημερώνει τον αντίπαλο μέσω του Log.


Τεχνολογίες για την υλοποίηση
1. Backend (Εξυπηρετητής & Λογική)
PHP: Είναι η γλώσσα στην οποία γράφτηκε όλο το Web API. Διαχειρίζεται τους κανόνες του παιχνιδιού, τον έλεγχο της σειράς των παικτών και τον υπολογισμό του σκορ.

MySQL: Η βάση δεδομένων όπου αποθηκεύονται τα πάντα: η κατάσταση του παιχνιδιού, τα στοιχεία των 52 φύλλων, τα σκορ και το ιστορικό κινήσεων (log).

2. Frontend (Διεπαφή Χρήστη - GUI)
HTML5 & CSS3: Χρησιμοποιήθηκαν για τη δομή και τη γραφική εμφάνιση του ταμπλό. Με το CSS δώσαμε μορφή στις κάρτες και υλοποιήσαμε τη στοίβαξή τους στο τραπέζι.

JavaScript & jQuery: Χρησιμοποιήθηκαν για να γίνει η σελίδα διαδραστική. Επιτρέπουν στον παίκτη να ρίχνει κάρτες με ένα κλικ και εμφανίζουν τα μηνύματα επιβεβαίωσης (π.χ. στην εγκατάλειψη).

3. Πρωτόκολλα Επικοινωνίας
JSON: Η μορφή στην οποία ανταλλάσσονται τα δεδομένα μεταξύ server και client. Το API στέλνει την κατάσταση του παιχνιδιού σε JSON μορφή και η JavaScript την "διαβάζει" για να ενημερώσει το GUI.

HTTP (POST/GET): Οι μέθοδοι με τις οποίες στέλνονται τα αιτήματα (π.χ. POST για τη ρίψη κάρτας, GET για τον έλεγχο της κατάστασης).


API:
1. Αυθεντικοποίηση Χρήστη
URL: http://localhost/xeri_project/api/auth.php

Λειτουργία: Δέχεται το username μέσω POST.

Περιγραφή: Δημιουργεί ένα μοναδικό token (τυχαία συμβολοσειρά) και το αποθηκεύει στη βάση δεδομένων για τον συγκεκριμένο παίκτη. Αυτό το token χρησιμοποιείται από το frontend για την ταυτοποίηση του παίκτη σε κάθε επόμενη κίνηση.

2. Δημιουργία ή Σύνδεση σε Παιχνίδι
URL: http://localhost/xeri_project/api/game_admin.php

Λειτουργία: Δέχεται τις παραμέτρους action (create, join, forfeit), username και game_id.

Περιγραφή:

Action create: Δημιουργεί ένα νέο παιχνίδι στη βάση με κατάσταση "waiting" και αποδίδει στον παίκτη την πλευρά "P1".

Action join: Επιτρέπει στον δεύτερο παίκτη να συνδεθεί στο παιχνίδι, ορίζοντάς τον ως "P2", και αλλάζει την κατάσταση του board σε "playing".

Action forfeit: Καταγράφει την εγκατάλειψη ενός παίκτη στο log και θέτει το status του παιχνιδιού σε "ended".

3. Έλεγχος Κατάστασης & Deadlock
URL: http://localhost/xeri_project/api/game_status.php

Λειτουργία: Δέχεται game_id και token μέσω GET.

Περιγραφή: Επιστρέφει την τρέχουσα εικόνα του παιχνιδιού σε JSON. Περιλαμβάνει τα φύλλα στο τραπέζι, τα φύλλα στο χέρι του παίκτη, το σκορ και τα τελευταία μηνύματα από το log. Επίσης, ελέγχει αν η τράπουλα και τα χέρια άδειασαν (Deadlock), ώστε να τερματίσει το παιχνίδι και να δώσει το bonus των +3 πόντων.


4. Εκτέλεση Κίνησης 
URL: http://localhost/xeri_project/api/play_card.php

Λειτουργία: Δέχεται card_id, token και game_id μέσω POST.

Περιγραφή: Ελέγχει αν είναι η σειρά του παίκτη και αν η κάρτα που έριξε "μαζεύει" φύλλα από το τραπέζι ή κάνει "Ξερή". Ενημερώνει το σκορ, τη συλλογή καρτών και το ιστορικό κινήσεων (log), ενώ αλλάζει τη σειρά στον αντίπαλο.


5. Αρχικοποίηση Τράπουλας
URL:  http://localhost/xeri_project/api/start_game.php

Λειτουργία: Καλείται αυτόματα από το game_admin.php κατά τη σύνδεση του δεύτερου παίκτη.

Περιγραφή: Δημιουργεί τις 52 εγγραφές καρτών στη MySQL, τις ανακατεύει και μοιράζει τα πρώτα φύλλα στους παίκτες (6 στον καθένα) και στο τραπέζι (4 φύλλα).

6. Αποσύνδεση
URL: http://localhost/xeri_project/api/logout.php

Λειτουργία: Δέχεται το token μέσω POST.

Περιγραφή: Διαγράφει τον παίκτη (ή ακυρώνει το token του) από τη βάση δεδομένων, τερματίζοντας την ενεργή του συνεδρία.
